Для запуска приожения необходимо перейти в директорию с файлом test-task-0.0.1-SNAPSHOT.jar в консоли и выполнить запуск следующей командой:

java -jar test-task-0.0.1-SNAPSHOT.jar --base_dir=<путь к базовой директории>

Когда сервер поднимется, можно отправлять ему запросы на http://localhost:8080/ в формате, указанном ниже.

Например: http://localhost:8080/get_topics?empty=n

Формат запросов (GET):

- Список топиков, данные по которым есть данные в этой директории (/get_topics?empty=n)
	Формат запроса:
	/get_topics[?empty={{y|yes|true}|{n|no|false}|<any other symbol>}] - флаг empty определяет, будут искаться пустые, непустые или все топики
- По каждому топику, таймстемп последнего запуска (/get_time_stamps?pos=last)
	Формат запроса:
	/get_time_stamps[?pos={{last|newest}|{first|oldest}|<any other symbol>}] - флаг pos определяет, будут показаны только последние, только первые или все таймстампы
- По каждому топику, для последнего запуска, статистику: суммарное число сообщений во всех патрициях, максимальное/минимальное число сообщений, среднее число сообщений (/get_stats?stats=sum,min,max,avg&pos=last)
	Формат запроса:
	/get_stats?stats={total|sum|ttl}[,{min|minimum}][,{max|maximum}][,{avg|average}][&pos={{last|newest}|{first|oldest}|<any other symbol>}]
- По каждому топику, список партиций и число сообщений по каждой партиции во время последнего запуска (/get_offsets?pos=last)
	Формат запроса:
	/get_offsets[?pos={{last|newest}|{first|oldest}|<any other symbol>}]


При создании программы сделаны следующие предположения (на основе вопросов, которые были заданы в начале разработки):

1.	Кириллические имена папок по умолчанию не поддерживаются
2.	Партиции могут повторяться в файле
3.	Пустые значения в файле не поддерживаются
4.	Запросы должны быть в формате описанном выше
5.	Ответ приходит в формате JSON
6.	«- По каждому топику» - значит для всех топиков
7.	«- Список топиков, данные по которым есть данные в этой директории» - имеются ввиду пустые файлы
8.	Внутри base_dir должна быть именно такая структура файлов, как описана в задании
9.	Для программы использовалась версия Spring Boot 1.2.5.RELEASE
10.	Особые сторонние библиотеки не использовались
11.	Для каждого сценария, описанного в задании, использовался отдельный запрос
